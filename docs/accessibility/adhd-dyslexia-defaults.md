# ADHD and Dyslexia Defaults

## Purpose

This file defines the **default** product behavior for accessibility-sensitive use.

The goal is not to build a separate “special mode” and forget about it.
The goal is to make the core experience calmer, clearer, and easier to sustain.

---

## 1. Default assumptions

Assume some solo learners may:
- struggle with attention shifts
- lose momentum after interruptions
- feel overwhelmed by backlog
- read more slowly
- depend more on audio than dense visual reading
- benefit from spacing and cleaner layout
- need shorter sessions and faster wins

---

## 2. ADHD-friendly defaults

### Session defaults
- default suggested session: 5 minutes
- break longer work into missions
- show remaining steps clearly
- allow easy stop-and-resume

### Home screen defaults
- one obvious primary action
- limited high-emphasis cards
- minimal simultaneous choices

### Feedback defaults
- fast positive reinforcement
- calm recovery messages
- clear expectations before each mission

### Routine defaults
- reminders linked to routine moments
- easy “do a tiny session now” action
- recovery mode after missed days

### Focus defaults
- low-clutter layout
- reduced unnecessary motion
- minimal visual competition
- optional focus mode that hides nonessential cards

---

## 3. Dyslexia-friendly defaults

### Text presentation
- larger default Arabic text option available quickly
- adjustable line spacing
- adjustable word spacing where technically possible
- avoid cramped card layouts
- avoid dense paragraph instructions

### UI text
- short labels
- short instructions
- clear hierarchy
- avoid decorative overload

### Multimodal support
- audio-first pathways easy to access
- synchronized highlighting during playback
- record-and-compare workflow
- visible chunking of memorization units

### Important principle
Do not rely on one “dyslexia font” as the main answer.
Customization is more reliable.

---

## 4. Arabic-script considerations

Arabic reading can create extra visual load because of:
- dense connected script
- multiple letter shapes
- diacritics
- crowding effects
- page-density issues

Design consequences:
- preserve clean spacing where possible
- do not shrink core memorization text unnecessarily
- let the learner reduce visual clutter
- support audio as a first-class aid
- keep mushaf display readable and stable

---

## 5. Recommended settings to expose early

Put these in an easy-access settings area:

- text size
- line spacing
- word spacing if supported
- contrast / theme
- reduce motion
- autoplay recitation
- repeat count
- default session length
- reminder window
- focus mode on/off

Do not bury these too deeply.

---

## 6. Suggested default values

### Good starting defaults
- default mission: 5 min
- low animation
- calm sound effects or optional mute
- medium-large text
- moderate spacing
- audio replay easy to reach
- no shame-based loss messages
- supportive recovery wording

---

## 7. Notification rules

Notifications should help routine, not create guilt.

Recommended style:
- “Ready for a 5-minute review?”
- “A small warm-up can protect yesterday’s memorization.”
- “Recovery mode is ready when you are.”

Avoid:
- “You failed”
- “Your streak is dying”
- “You are behind”

---

## 8. Error and failure language

Use:
- fragile
- due
- ready
- repair
- recover
- strengthen

Avoid heavy language like:
- broken
- failed learner
- lost progress
- disaster
- ruined streak

---

## 9. Interaction rules

Make these interactions easy:
- replay audio
- reveal hint
- mark self-grade
- save note
- continue next step
- pause and resume

Avoid requiring many taps before the actual memory work begins.

---

## 10. Accessibility success test

A good test for the product is this:

Can a tired, distracted solo learner open the app, do one useful thing in under a minute, and leave feeling clearer rather than worse?

If not, the UX still needs improvement.

---

## 11. Final summary

Accessibility here is not a compliance checkbox.
It is directly tied to whether the memorization system will be sustainable in real life.

For this product, accessibility means:
- lower friction
- less overwhelm
- better recovery
- more audio support
- cleaner reading surfaces
- calmer daily guidance
